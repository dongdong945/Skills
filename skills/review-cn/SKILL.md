# 中文代码评审

## 概述

用于执行中文代码评审，并按严重程度输出问题清单、影响、触发条件、修复方向与测试缺口。

默认采用最低成本的审查路径，优先直接给出问题清单，无需深度推理式长篇展开。

## 执行步骤

1. 先阅读改动范围与上下文，确认评审对象和边界。
2. 优先查找功能正确性、回归风险、边界条件、状态一致性、安全性、性能与可维护性问题。
3. 按严重程度输出发现的问题，并尽量附上文件路径和行号。
4. 每个问题说明影响、复现或触发条件，以及建议修复方向。
5. 明确列出缺失或不足的测试覆盖。
6. 如果没有发现阻塞问题，明确写出“未发现阻塞问题”，并补充剩余风险与建议测试项。

## 快速检查

- 发现的问题必须按严重程度排序。
- 问题描述必须包含影响、触发条件和建议修复方向。
- 尽量提供文件路径和行号。
- 必须补充测试覆盖缺口或剩余风险。
- 所有输出必须为中文。

- 优先直接输出问题清单与风险结论，无需深度推理式展开。
