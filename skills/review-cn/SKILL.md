# Review CN

执行代码评审时遵循以下规则：

1. 先给出发现的问题，按严重程度排序。
2. 每个问题包含：影响、复现或触发条件、建议修复方向。
3. 尽量提供文件路径和行号。
4. 重点关注：功能正确性、回归风险、边界条件、并发/状态一致性、安全性、性能与可维护性。
5. 明确列出缺失或不足的测试覆盖。
6. 如果没有发现问题，明确写出“未发现阻塞问题”，并给出剩余风险与建议测试项。
7. 所有输出必须为中文。
